⛏ Mining Guide

How to Mine Tensorium (TXM)

Step-by-step guide to setting up a node, creating a wallet, and mining TXM on the public testnet. No GPU required — CPU mining is available now.

🌐 Network: tensorium-testnet-0 ⚙ Difficulty: 26 bits 🎁 Reward: 15.23557865 TXM/block 💻 Platform: Linux x86_64
⚠
Testnet only. Tensorium is currently in public testnet. Testnet TXM has no monetary value. The chain may be reset before mainnet. Mine to test the software, report bugs, and participate in the community.
4 steps to start mining
From zero to mining in under 5 minutes on any Linux machine.

Install
One-line installer downloads all binaries and sets up your node.
02

Generate a secure wallet and get your TXM address.
03

2 Installation

2.1 One-line Installer (Recommended)

The installer downloads pre-compiled binaries, creates a wallet, syncs the chain, and optionally sets up systemd services:

curl -fsSL https://raw.githubusercontent.com/rygroup-dev/tensorium-core/main/install.sh | bash

The installer will:

  1. Download tensorium-node, txmminer, txmwallet to /usr/local/bin/
  2. Prompt you for a wallet passphrase and create a wallet at ~/tensorium-node/wallet.json
  3. Initialize the chain (genesis block) and sync from the seed node
  4. Optionally install systemd services for auto-start on reboot

2.2 Manual Installation

Download the latest binaries from GitHub Releases:

# Download
RELEASE="v0.3.1-mainnet-candidate"
BASE="https://github.com/rygroup-dev/tensorium-core/releases/download/${RELEASE}"

curl -fsSL -o tensorium-node  "${BASE}/tensorium-node-linux-x86_64"
curl -fsSL -o txmminer         "${BASE}/txmminer-linux-x86_64"
curl -fsSL -o txmwallet        "${BASE}/txmwallet-linux-x86_64"

# Make executable and install
chmod +x tensorium-node txmminer txmwallet
sudo mv tensorium-node txmminer txmwallet /usr/local/bin/

# Verify
tensorium-node --help

2.3 Build from Source

If you prefer to build from source (requires Rust ≥ 1.76):

git clone https://github.com/rygroup-dev/tensorium-core.git
cd tensorium-core
cargo build --release

# Binaries will be in target/release/
sudo cp target/release/tensorium-node target/release/txmminer target/release/txmwallet /usr/local/bin/

3 Create a Wallet

3.1 Create New Wallet

# Set where to store the wallet
export TENSORIUM_WALLET=~/tensorium-node/wallet.json

# Set your passphrase (use a strong passphrase — this encrypts your private key)
export TENSORIUM_WALLET_PASSPHRASE="your-strong-passphrase-here"

# Create the wallet
txmwallet create

This generates a secp256k1 keypair and saves it encrypted with Argon2id + ChaCha20-Poly1305. Output example:

address=txm1abc123...
public_key=02ab34...
wallet_version=2
encrypted=true

3.2 Get Your Mining Address

export TENSORIUM_WALLET=~/tensorium-node/wallet.json
export TENSORIUM_WALLET_PASSPHRASE="your-passphrase"

txmwallet getnewaddress
# → txm1abc123xyz...   (your TXM address)

Save this address — you will provide it to txmminer so block rewards go to your wallet.

3.3 Backup Your Wallet

🔑
Back up your wallet file immediately. If you lose wallet.json or forget your passphrase, your funds are permanently unrecoverable. Copy the wallet file to at least two separate locations (external drive, cloud storage, etc.). 
# Backup
cp ~/tensorium-node/wallet.json ~/wallet-backup-$(date +%Y%m%d).json

# Verify backup can be decrypted
TENSORIUM_WALLET=~/wallet-backup-*.json \
TENSORIUM_WALLET_PASSPHRASE="your-passphrase" \
txmwallet unlock-check

3.4 Check Balance

export TENSORIUM_WALLET=~/tensorium-node/wallet.json
export TENSORIUM_WALLET_PASSPHRASE="your-passphrase"
export TENSORIUM_STATE=~/tensorium-node/state.json

txmwallet balance

Note: Coinbase rewards (block rewards) require 100 confirmations before they appear as spendable.

4 Run a Node

4.1 Initialize the Chain

mkdir -p ~/tensorium-node
cd ~/tensorium-node

# Initialize — this mines the genesis block (~30 seconds at difficulty 26)
TENSORIUM_STATE=~/tensorium-node/state.json \
TENSORIUM_MEMPOOL=~/tensorium-node/mempool.json \
TENSORIUM_BANS=~/tensorium-node/banlist.json \
tensorium-node init

4.2 Sync from Seed Node

TENSORIUM_STATE=~/tensorium-node/state.json \
TENSORIUM_MEMPOOL=~/tensorium-node/mempool.json \
TENSORIUM_BANS=~/tensorium-node/banlist.json \
tensorium-node sync 157.230.44.162:23333

Output shows sync progress:

sync start: peer=157.230.44.162:23333 peer_height=150 our_height=0
  synced +50 blocks  height=50  total_synced=50
  synced +50 blocks  height=100  total_synced=100
  synced +50 blocks  height=150  total_synced=150
sync complete: tip=150 synced=150 blocks from 157.230.44.162:23333

4.3 Start RPC and P2P

Open two terminal windows (or use nohup / tmux):

# Terminal 1 — RPC server (do NOT expose this port publicly)
TENSORIUM_STATE=~/tensorium-node/state.json \
TENSORIUM_MEMPOOL=~/tensorium-node/mempool.json \
TENSORIUM_BANS=~/tensorium-node/banlist.json \
TENSORIUM_PEERS=157.230.44.162:23333 \
tensorium-node rpc 127.0.0.1:23332

# Terminal 2 — P2P listener (this port should be open/forwarded)
TENSORIUM_STATE=~/tensorium-node/state.json \
TENSORIUM_MEMPOOL=~/tensorium-node/mempool.json \
TENSORIUM_BANS=~/tensorium-node/banlist.json \
tensorium-node p2p-listen 0.0.0.0:23333

Check your node is working:

curl -s http://localhost:23332/getblockcount
# → {"blocks":151,"chain_id":"tensorium-testnet-0","height":150}

4.4 Systemd Service (Auto-start on Reboot)

sudo tee /etc/systemd/system/tensorium-rpc.service <<EOF
[Unit]
Description=Tensorium Node RPC
After=network.target

[Service]
Type=simple
User=$(whoami)
Environment=TENSORIUM_STATE=$HOME/tensorium-node/state.json
Environment=TENSORIUM_MEMPOOL=$HOME/tensorium-node/mempool.json
Environment=TENSORIUM_BANS=$HOME/tensorium-node/banlist.json
Environment=TENSORIUM_PEERS=157.230.44.162:23333
ExecStart=/usr/local/bin/tensorium-node rpc 127.0.0.1:23332
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

sudo tee /etc/systemd/system/tensorium-p2p.service <<EOF
[Unit]
Description=Tensorium Node P2P
After=network.target

[Service]
Type=simple
User=$(whoami)
Environment=TENSORIUM_STATE=$HOME/tensorium-node/state.json
Environment=TENSORIUM_MEMPOOL=$HOME/tensorium-node/mempool.json
Environment=TENSORIUM_BANS=$HOME/tensorium-node/banlist.json
ExecStart=/usr/local/bin/tensorium-node p2p-listen 0.0.0.0:23333
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now tensorium-rpc tensorium-p2p

5 Start Mining

5.1 Basic Mining

YOUR_ADDRESS="txm1your_address_here"

# Mine continuously — auto-detects CPU thread count
txmminer 127.0.0.1:23332 "$YOUR_ADDRESS"

Sample output:

txmminer  rpc=127.0.0.1:23332  miner=txm1abc...  threads=8
Press Ctrl+C to stop.

mining  height=151  bits=26  threads=8  … ✓  height=151  nonce=8432156  7.2s  9.31 MH/s
mining  height=152  bits=26  threads=8  … ✓  height=152  nonce=52341   0.1s  8.87 MH/s
mining  height=153  bits=26  threads=8  … ✓  height=153  nonce=71234812 61.3s 9.14 MH/s

5.2 Specify Thread Count

# Use 4 threads
txmminer 127.0.0.1:23332 "$YOUR_ADDRESS" 4

# Use all available CPU cores (default)
txmminer 127.0.0.1:23332 "$YOUR_ADDRESS"
💡
More threads = more hashrate. Each thread searches a different portion of the nonce space in parallel. On an 8-core CPU, using 8 threads gives ~8× the hashrate of a single thread. Leave 1-2 cores free for system tasks if needed.

⚠
High CPU hashrate ≠ fair. At testnet difficulty 26, a powerful server can mine blocks in milliseconds. This is intentional — testnet is for testing the chain, not fair distribution. Difficulty will auto-adjust upward as more hashrate joins, or will be reset with higher initial difficulty before mainnet.

5.4 Keep Miner Running

# Using nohup
nohup txmminer 127.0.0.1:23332 "$YOUR_ADDRESS" > ~/tensorium-node/miner.log 2>&1 &
echo "Miner PID: $!"

# Using tmux
tmux new -s miner "txmminer 127.0.0.1:23332 $YOUR_ADDRESS"

# Systemd service
sudo tee /etc/systemd/system/tensorium-miner.service <<EOF
[Unit]
Description=Tensorium CPU Miner
After=tensorium-rpc.service

[Service]
Type=simple
User=$(whoami)
ExecStart=/usr/local/bin/txmminer 127.0.0.1:23332 YOUR_TXM_ADDRESS
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now tensorium-miner

6.2 Coinbase Maturity

Block rewards cannot be spent until 100 blocks have been built on top of the block you mined. This protects against reorg invalidating your reward. At 60s block time, maturity takes ~100 minutes.

export TENSORIUM_WALLET_PASSPHRASE="your-passphrase"
export TENSORIUM_STATE=~/tensorium-node/state.json

# Build and sign a transaction (100000000 atoms = 1.0 TXM)
txmwallet send txm1recipient_address 100000000 /tmp/my-tx.json

# Broadcast to your node
txmwallet broadcast /tmp/my-tx.json 127.0.0.1:23332

7.4 Node Commands Reference

tensorium-node init                      # Initialize genesis block
tensorium-node status                    # Show chain tip and height
tensorium-node rpc [bind]                # Start HTTP RPC (default 127.0.0.1:23332)
tensorium-node p2p-listen [bind]         # Start P2P server (default 0.0.0.0:23333)
tensorium-node sync [host:port]          # Download missing blocks from a peer
tensorium-node p2p-connect <host:port>  # Diagnostic handshake to a peer
tensorium-node peers                     # Print configured peers
tensorium-node banlist                   # Show peer ban list
tensorium-node unban <ip>               # Remove a ban

— FAQ

Is testnet TXM worth anything? +
No. Testnet TXM has zero monetary value. Testnet exists to test the protocol and find bugs before mainnet. The chain may be reset. Do not buy, sell, or trade testnet coins.
Do I need a GPU to mine? +
Not on testnet. Current testnet difficulty (26 bits) is designed for CPU mining. Any modern CPU can mine blocks. For mainnet, GPU mining will be required (difficulty 36+ bits). We will release a CUDA miner before mainnet.
Why does genesis init take so long? +
Genesis initialization mines the first block at difficulty 26, which requires finding a nonce that produces a hash with 26 leading zero bits (~67 million hash attempts on average). On a basic VPS, this takes 1–3 minutes. This is intentional — all nodes must mine the same genesis with the same fixed timestamp to get the same block hash.
My miner found a block but txmwallet balance shows 0. Why? +
Coinbase rewards (block rewards) require 100 confirmations before they are spendable. Wait for 100 more blocks to be mined after your block, then check your balance again.
What is the seed node and do I need to keep it connected? +
The seed node at 157.230.44.162:23333 is the bootstrap node run by the Tensorium team. You need it to sync your initial chain state and to broadcast your mined blocks. Set TENSORIUM_PEERS=157.230.44.162:23333 in your RPC service for automatic block propagation.
Can I run the node and miner on different machines? +
Yes. The miner connects to the node via RPC. If the miner is on a different machine, bind the RPC to 0.0.0.0 instead of 127.0.0.1, and use a firewall to restrict RPC access to trusted IPs only. Never expose the RPC port publicly without authentication (not yet implemented).
I found a bug. Where do I report it? +
Open an issue on GitHub: github.com/rygroup-dev/tensorium-core/issues. Include your OS version, the binary version (from release tag), and steps to reproduce.
When is mainnet? +
No date is set. Mainnet requires: stable GPU testnet running for ≥4 weeks, independent security audit, GPU miner published, and final community review. See the Roadmap for full details. We will not rush mainnet.
