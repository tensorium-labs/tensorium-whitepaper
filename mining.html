How to Mine Tensorium (TXM)

Step-by-step guide to setting up a node, creating a wallet, and mining TXM on the public testnet. The current onboarding reset is CPU-friendly, while mainnet-candidate remains GPU-first.

🌐 Network: tensorium-mainnet-candidate-0 ⚙ Difficulty: 20 bits 🎁 Reward: 15.23557865 TXM/block 💻 Platform: Linux x86_64
⚠
Testnet only. Tensorium is currently in public testnet. Testnet TXM has no monetary value. The chain may be reset before mainnet. The public testnet was reset on 2026-06-01 to easier CPU-minable settings for onboarding, faucet funding, and wallet testing.
4 steps to start mining
From zero to mining in under 5 minutes on any Linux machine.

2 Installation

2.1 One-line Installer (Recommended)

The installer downloads pre-compiled binaries, creates a wallet, syncs the chain, and optionally sets up systemd services:

curl -fsSL https://raw.githubusercontent.com/tensorium-labs/tensorium-core/main/install.sh | bash

The installer will:

  1. Download tensorium-node, txmminer, txmwallet to /usr/local/bin/
  2. Prompt you for a wallet passphrase and create a wallet at ~/tensorium-node/wallet.json
  3. Initialize the chain (genesis block) and sync from the seed node
  4. Optionally install systemd services for auto-start on reboot

2.2 Manual Installation

Download the latest testnet binaries from GitHub Releases:

# Download
RELEASE="v0.2.0-testnet"
BASE="https://github.com/tensorium-labs/tensorium-core/releases/download/${RELEASE}"

curl -fsSL -o tensorium-node  "${BASE}/tensorium-node-linux-x86_64"
curl -fsSL -o txmminer         "${BASE}/txmminer-linux-x86_64"
curl -fsSL -o txmwallet        "${BASE}/txmwallet-linux-x86_64"

# Make executable and install
chmod +x tensorium-node txmminer txmwallet
sudo mv tensorium-node txmminer txmwallet /usr/local/bin/

# Verify
tensorium-node --help

2.3 Build from Source

If you prefer to build from source (requires Rust ≥ 1.76):

git clone https://github.com/tensorium-labs/tensorium-core.git
cd tensorium-core
cargo build --release

# Binaries will be in target/release/
sudo cp target/release/tensorium-node target/release/txmminer target/release/txmwallet /usr/local/bin/

3 Create a Wallet

3.1 Create New Wallet

# Set where to store the wallet
export TENSORIUM_WALLET=~/tensorium-node/wallet.json

# Set your passphrase (use a strong passphrase — this encrypts your private key)
export TENSORIUM_WALLET_PASSPHRASE="your-strong-passphrase-here"

# Create the wallet
txmwallet create

This generates a secp256k1 keypair and saves it encrypted with Argon2id + ChaCha20-Poly1305. Output example:

address=txm1abc123...
public_key=02ab34...
wallet_version=2
encrypted=true

3.2 Get Your Mining Address

export TENSORIUM_WALLET=~/tensorium-node/wallet.json
export TENSORIUM_WALLET_PASSPHRASE="your-passphrase"

txmwallet getnewaddress
# → txm1abc123xyz...   (your TXM address)

Save this address — you will provide it to txmminer so block rewards go to your wallet.

3.3 Backup Your Wallet

🔑
Back up your wallet file immediately. If you lose wallet.json or forget your passphrase, your funds are permanently unrecoverable. Copy the wallet file to at least two separate locations (external drive, cloud storage, etc.). 
# Backup
cp ~/tensorium-node/wallet.json ~/wallet-backup-$(date +%Y%m%d).json

# Verify backup can be decrypted
TENSORIUM_WALLET=~/wallet-backup-*.json \
TENSORIUM_WALLET_PASSPHRASE="your-passphrase" \
txmwallet unlock-check

3.4 Check Balance

export TENSORIUM_WALLET=~/tensorium-node/wallet.json
export TENSORIUM_WALLET_PASSPHRASE="your-passphrase"
export TENSORIUM_STATE=~/tensorium-node/state.json

txmwallet balance

Note: On the current onboarding testnet, coinbase rewards require 10 confirmations before they appear as spendable.

4 Run a Node

4.1 Initialize the Chain

mkdir -p ~/tensorium-node
cd ~/tensorium-node

# Initialize — current public testnet genesis is hardcoded, so init is instant
TENSORIUM_STATE=~/tensorium-node/state.json \
TENSORIUM_MEMPOOL=~/tensorium-node/mempool.json \
TENSORIUM_BANS=~/tensorium-node/banlist.json \
tensorium-node mainnet-candidate init

4.2 Sync from Seed Node

TENSORIUM_STATE=~/tensorium-node/state.json \
TENSORIUM_MEMPOOL=~/tensorium-node/mempool.json \
TENSORIUM_BANS=~/tensorium-node/banlist.json \
tensorium-node mainnet-candidate sync seed.tensoriumlabs.com:33333

Output shows sync progress:

sync start: peer=seed.tensoriumlabs.com:33333 peer_height=150 our_height=0
  synced +50 blocks  height=50  total_synced=50
  synced +50 blocks  height=100  total_synced=100
  synced +50 blocks  height=150  total_synced=150
sync complete: tip=150 synced=150 blocks from seed.tensoriumlabs.com:33333

4.3 Start RPC and P2P

Open two terminal windows (or use nohup / tmux):

# Terminal 1 — RPC server (do NOT expose this port publicly)
TENSORIUM_STATE=~/tensorium-node/state.json \
TENSORIUM_MEMPOOL=~/tensorium-node/mempool.json \
TENSORIUM_BANS=~/tensorium-node/banlist.json \
TENSORIUM_PEERS=seed.tensoriumlabs.com:33333 \
tensorium-node mainnet-candidate rpc 127.0.0.1:33332

# Terminal 2 — P2P listener (this port should be open/forwarded)
TENSORIUM_STATE=~/tensorium-node/state.json \
TENSORIUM_MEMPOOL=~/tensorium-node/mempool.json \
TENSORIUM_BANS=~/tensorium-node/banlist.json \
tensorium-node mainnet-candidate p2p-listen 0.0.0.0:33333

Check your node is working:

curl -s http://localhost:33332/getblockcount
# → {"blocks":151,"chain_id":"tensorium-mainnet-candidate-0","height":150}

4.4 Systemd Service (Auto-start on Reboot)

sudo tee /etc/systemd/system/tensorium-rpc.service <<EOF
[Unit]
Description=Tensorium Node RPC
After=network.target

[Service]
Type=simple
User=$(whoami)
Environment=TENSORIUM_STATE=$HOME/tensorium-node/state.json
Environment=TENSORIUM_MEMPOOL=$HOME/tensorium-node/mempool.json
Environment=TENSORIUM_BANS=$HOME/tensorium-node/banlist.json
Environment=TENSORIUM_PEERS=seed.tensoriumlabs.com:33333
ExecStart=/usr/local/bin/tensorium-node mainnet-candidate rpc 127.0.0.1:33332
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

sudo tee /etc/systemd/system/tensorium-p2p.service <<EOF
[Unit]
Description=Tensorium Node P2P
After=network.target

[Service]
Type=simple
User=$(whoami)
Environment=TENSORIUM_STATE=$HOME/tensorium-node/state.json
Environment=TENSORIUM_MEMPOOL=$HOME/tensorium-node/mempool.json
Environment=TENSORIUM_BANS=$HOME/tensorium-node/banlist.json
ExecStart=/usr/local/bin/tensorium-node mainnet-candidate p2p-listen 0.0.0.0:33333
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now tensorium-rpc tensorium-p2p

5 Start Mining

5.1 Basic Mining

YOUR_ADDRESS="txm1your_address_here"

# Mine continuously — auto-detects CPU thread count
txmminer 127.0.0.1:33332 "$YOUR_ADDRESS"

Sample output:

txmminer  rpc=127.0.0.1:33332  miner=txm1abc...  threads=8
Press Ctrl+C to stop.

mining  height=151  bits=20  threads=8  … ✓  height=151  nonce=8432156  0.2s  9.31 MH/s
mining  height=152  bits=20  threads=8  … ✓  height=152  nonce=52341   0.0s  8.87 MH/s
mining  height=153  bits=20  threads=8  … ✓  height=153  nonce=71234812  1.0s  9.14 MH/s

5.2 Specify Thread Count

# Use 4 threads
txmminer 127.0.0.1:33332 "$YOUR_ADDRESS" 4

# Use all available CPU cores (default)
txmminer 127.0.0.1:33332 "$YOUR_ADDRESS"
💡
More threads = more hashrate. Each thread searches a different portion of the nonce space in parallel. On an 8-core CPU, using 8 threads gives ~8× the hashrate of a single thread. Leave 1-2 cores free for system tasks if needed.

⚠
High CPU hashrate ≠ fair. At the current 20-bit onboarding difficulty, a powerful server can mine blocks extremely quickly. This is intentional for testing, faucet replenishment, and wallet onboarding. Fairer GPU-first conditions are reserved for mainnet-candidate and later launch stages.

5.4 Keep Miner Running

# Using nohup
nohup txmminer 127.0.0.1:33332 "$YOUR_ADDRESS" > ~/tensorium-node/miner.log 2>&1 &
echo "Miner PID: $!"

# Using tmux
tmux new -s miner "txmminer 127.0.0.1:33332 $YOUR_ADDRESS"

# Systemd service
sudo tee /etc/systemd/system/tensorium-miner.service <<EOF
[Unit]
Description=Tensorium CPU Miner
After=tensorium-rpc.service

[Service]
Type=simple
User=$(whoami)
ExecStart=/usr/local/bin/txmminer 127.0.0.1:33332 YOUR_TXM_ADDRESS
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now tensorium-miner

export TENSORIUM_WALLET_PASSPHRASE="your-passphrase"
export TENSORIUM_STATE=~/tensorium-node/state.json

# Build and sign a transaction (100000000 atoms = 1.0 TXM)
txmwallet send txm1recipient_address 100000000 /tmp/my-tx.json

# Broadcast to your node
txmwallet broadcast /tmp/my-tx.json 127.0.0.1:33332

7.4 Node Commands Reference

tensorium-node mainnet-candidate init                      # Initialize genesis block
tensorium-node status                    # Show chain tip and height
tensorium-node mainnet-candidate rpc [bind]                # Start HTTP RPC (default 127.0.0.1:33332)
tensorium-node mainnet-candidate p2p-listen [bind]         # Start P2P server (default 0.0.0.0:33333)
tensorium-node mainnet-candidate sync [host:port]          # Download missing blocks from a peer

tensorium-node banlist                   # Show peer ban list
tensorium-node unban <ip>               # Remove a ban

— FAQ

Is testnet TXM worth anything? +
No. Testnet TXM has zero monetary value. Testnet exists to test the protocol and find bugs before mainnet. The chain may be reset. Do not buy, sell, or trade testnet coins.
Do I need a GPU to mine? +
Not on the current public testnet. The onboarding reset lowered difficulty to 20 bits specifically so ordinary CPUs can mine and test the wallet or faucet flow. Mainnet-candidate remains GPU-first with much higher difficulty.
Why does genesis init take so long? +
It should not anymore on the current public testnet. The genesis block is now hardcoded for fast onboarding, so tensorium-node mainnet-candidate init should be effectively instant. If init is still slow, you are probably on an older binary or older chain mode.
My miner found a block but txmwallet balance shows 0. Why? +
On the current onboarding testnet, coinbase rewards require 10 confirmations before they are spendable. Wait for 10 more blocks to be mined after your block, then check your balance again.
What is the seed node and do I need to keep it connected? +
The seed node at seed.tensoriumlabs.com:33333 is the bootstrap node run by the Tensorium team. You need it to sync your initial chain state and to broadcast your mined blocks. Set TENSORIUM_PEERS=seed.tensoriumlabs.com:33333 in your RPC service for automatic block propagation.
Can I run the node and miner on different machines? +
Yes. The miner connects to the node via RPC. If the miner is on a different machine, bind the RPC to 0.0.0.0 instead of 127.0.0.1, and use a firewall to restrict RPC access to trusted IPs only. Never expose the RPC port publicly without authentication (not yet implemented).
I found a bug. Where do I report it? +
Open an issue on GitHub: github.com/tensorium-labs/tensorium-core/issues. Include your OS version, the binary version (from release tag), and steps to reproduce.
When is mainnet? +
No date is set. Mainnet requires: stable GPU testnet running for ≥4 weeks, independent security audit, GPU miner published, and final community review. See the Roadmap for full details. We will not rush mainnet.
